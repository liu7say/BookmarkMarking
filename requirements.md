# 浏览器书签整理工具需求文档

## 1. 项目背景与目标
本项目旨在开发一款**浏览器书签整理与可视化管理工具**。用户可以通过手动上传从浏览器导出的书签 HTML 文件，系统将自动解析并在本地转换为结构化的树状 JSON 数据进行存储与展示。除了展示原有的书签全面信息外，系统还提供给用户自定义打标（Tagging）的能力，帮助用户更高效地管理和分类个人的信息资产。

## 2. 技术栈选型
*   **前端框架**：Vue.js (推荐使用 Vue 3 + Composition API 以获得更好的性能与可维护性)
*   **构建工具**：Vite (提供极速的冷启动与热更新)
*   **UI 规范**：Fluent 2 设计规范 (强调现代、轻盈、系统感强，使用亚克力、柔和渐变与半透明卡片)
*   **状态管理**：Pinia (管理树状数据流及用户标记数据)
*   **路由管理**：Vue Router (视后续页面复杂度决定是否引入)
*   **本地存储**：Dexie.js (封装了 IndexedDB，用于在浏览器端存储大容量的书签 JSON 数据与图标，替代 MySQL 等传统后端数据库，实现纯纯本地化与隐私安全)

## 3. 核心功能需求

### 3.1 书签上传与解析入口
*   **功能描述**：提供一个显眼、符合 Fluent 2 视觉规范的文件上传入口（例如拖拽上传区域或原生系统风格的上传按钮）。
*   **业务逻辑**：支持解析标准的浏览器导出的书签 HTML 文件（如 Chrome/Edge 导出的 favorites.html）。

### 3.2 HTML 到 JSON 的转换逻辑
*   **功能描述**：在浏览器前端完成 HTML 的 DOM 解析，并将其转换成深层树状 JSON 格式。
*   **信息提取**：
    *   `Title` (书签或文件夹标题)
    *   `URL` (书签链接地址)
    *   `Icon` (网站 favicon 或 Base64 图标数据)
    *   `Add Date` (书签添加时间)
    *   `Last Modified` (最后修改时间，如有)
    *   `Children` (子节点数组，用于构成树状结构)

### 3.3 书签树状展示与交互
*   **功能描述**：以树状列表或嵌套卡片的形式展示结构化的书签数据。
*   **交互要求**：
    *   文件夹的展开与折叠（附带 150-300ms 的平滑 Fluent 缓动动画）。
    *   书签卡片悬停 (Hover) 时，展示微弱的上浮感、轻微阴影加深或高光描边（Fluent 2 亚克力与深度效果）。

### 3.4 自定义打标 (Tagging) 能力
*   **功能描述**：允许用户对解析后的每一条具体书签（甚至文件夹）进行自定义标签管理。
*   **业务逻辑**：
    *   自定义标签的数据需要与原始树状 JSON 数据关联。
    *   **数据持久化**：使用 **IndexedDB (通过 Dexie.js)** 进行全量本地存储。这样不仅能轻松突破 `localStorage` 的 5MB 容量限制（完美应对包含大量 Base64 图标的书签数据），还能免去搭建 MySQL 等后端数据库的繁琐，真正做到“即开即用、数据完全掌握在自己手中”。

## 4. UI 与视觉设计要求 (基于 Fluent 2 规范)
*   **色彩与背景**：使用浅色渐变或柔和纹理作为全局背景。
*   **材质与深度**：核心内容区、弹窗或悬浮菜单采用「亚克力」材质（半透明底色 + 背景模糊），边缘伴随细腻的高光描边。
*   **圆角与排版**：使用一致的系统级圆角，图标采用干净的线性图标（Fluent Icons），文字排版层次分明。
*   **动效**：拒绝过度夸张的弹跳，点击、悬浮、切换状态等交互需配以流畅自然的缓动过渡（轻量的空间变化）。
